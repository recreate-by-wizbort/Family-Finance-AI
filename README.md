# Family Finance AI — Семейный финансовый помощник

**RECREATE BY WIZBORT**

Полнофункциональное SPA-приложение для управления личными и семейными финансами с интегрированным AI-ассистентом. Платформа имитирует мобильный банкинг с глубокой аналитикой расходов/доходов, семейным бюджетированием, целями накоплений и персональными AI-рекомендациями.

## Быстрый старт

```bash
cd family-finance-ai
npm install
```

Создайте файл `.env` на основе `.env.example` и добавьте API-ключи:

```env
VITE_GEMINI_API_KEY_1=ваш_ключ
VITE_OPENROUTER_API_KEY_1=ваш_ключ
```

Запуск dev-сервера:

```bash
npm run dev
```

Сборка для продакшена:

```bash
npm run build
npm run preview
```

## Ключевые функции

### Банковские операции
- Просмотр карт и счетов (HUMO, UZCARD, VISA) с реальными BIN-валидациями
- Мультивалютные счета (UZS, USD, EUR) с открытием/пополнением/выводом
- Вклады с расчётом процентов и графиками выплат
- Переводы между своими картами, P2P-переводы, избранные получатели
- Привязка внешних карт других банков
- Курсы валют ЦБ в реальном времени (кэширование)

### Финансовая аналитика (CostPage)
- Три режима: расходы (`debit`), мониторинг (`monitoring`), доходы (`credit`)
- Фильтрация по периодам: неделя / месяц / год / произвольный диапазон
- Фильтрация по счетам и участникам семьи
- Кольцевые диаграммы по категориям расходов
- Режим исключения переводов из расходов
- Полная история операций с детализацией
- Встраивание в страницу семейной группы

### Семейная группа
- Управление участниками семьи (владелец, супруга, ребёнок)
- Семейный резерв с пополнением от участников
- Визуализация вкладов каждого участника (donut-диаграмма)
- Распределение средств из резерва на цели
- Семейная аналитика расходов/доходов

### Финансовые цели
- Предустановленные цели: машина, отпуск, подушка безопасности, жильё
- Создание пользовательских целей
- Пополнение с карт и из семейного резерва
- Отслеживание прогресса
- «Сюрприз»-цели (скрытые от других участников)

### Спецпредложения
- Карточки промо-акций с описанием и условиями
- Микрозайм — демо-оффер с расчётом аннуитетного платежа
- Контекстные предложения в AI-чате при нехватке средств

## AI-ассистент

Интегрированный финансовый помощник, работающий с двумя LLM-провайдерами.

### Провайдеры и модели

| Провайдер | Модель | Тип |
|-----------|--------|-----|
| Google Gemini | gemini-2.5-flash | Streaming SSE |
| OpenRouter | nvidia/nemotron-3-super-120b-a12b:free | Streaming SSE |

### Как работает

1. **Пул ключей** — при старте приложения `initApiKeyPool()` проверяет все API-ключи из `.env` (до 20 штук на каждый провайдер). Нерабочие ключи исключаются.

2. **Чередование провайдеров** — каждый запрос чередует Gemini и OpenRouter (round-robin). Внутри провайдера ключи также ротируются. При ошибке — автоматический переход к следующему ключу/провайдеру.

3. **Контекст** — каждый запрос включает:
   - Системный промпт с ролью финансового помощника
   - Данные бюджета за выбранный период (баланс, доходы, расходы, топ категорий)
   - Помесячную динамику за 12 месяцев
   - Полную историю диалога (до 24 последних сообщений)

4. **Трёхуровневая обработка**:
   - **FAQ** (`mockDataAI.js`) — мгновенные ответы на типовые вопросы без вызова AI
   - **Аналитический движок** (`analysisEngine.js`) — rule-based ответы с расчётами из данных пользователя (50+ сценариев)
   - **LLM** (`api.js`) — стриминговый вызов Gemini/OpenRouter для сложных вопросов

5. **Интерактивные элементы** — AI генерирует кнопки действий `[КНОПКА: ...]` и навигационные ссылки `[НАВИГАЦИЯ: ... | /path]`, которые рендерятся как кликабельные элементы.

6. **Персистентность** — история чата, выбранный период и скоуп сохраняются в `localStorage`. Чат не теряется при навигации между страницами.

### Категории AI-чата

Двухуровневое дерево тем для быстрого доступа:

**Личные:**
- Расходы (еда, транспорт, подписки, коммунальные, маркетплейсы, здоровье, развлечения, одежда, образование)
- Доходы (зарплата, пассивный доход)
- Цели (машина, отпуск, подушка, жильё)
- Прогноз (до конца периода, следующий период)
- Оптимизация (где сэкономить, автоматизация, сравнение периодов)

**Семейные:**
- Обзор семьи (кто сколько тратит, переводы)
- Семейные цели (общие, сюрприз-цели)
- Семейный бюджет (резерв, оптимизация)
- Дети (расходы, лимиты, образование)

## Безопасность

- **PIN-авторизация** — 4-значный код при входе в приложение
- **Сессия 30 минут** — автоматическая блокировка при неактивности (`sessionLock.js`)
- **RequirePinRoute** — все защищённые маршруты проверяют наличие активной сессии
- **Валидация BIN** — проверка номеров карт по BIN-базе

## Данные

Приложение работает на демо-данных (`mockData.js`), имитирующих реальный банковский контракт:

- **Пользователь**: Андрей (user_1) — основной владелец
- **Семья**: Мадина (супруга), Тимур (сын)
- **Счета**: дебетовые, зарплатные, депозитные (UZS/USD/EUR)
- **Карты**: HUMO, UZCARD, VISA с реальными BIN-префиксами
- **Транзакции**: ~50+ записей по категориям с timestamps
- **Категории расходов**: продукты, кафе, транспорт, одежда, развлечения, маркетплейсы, здоровье, образование, коммунальные, подписки и др.

Пользовательские действия (открытие счетов, вкладов, переводы, пополнение целей) персистятся в `localStorage`.
